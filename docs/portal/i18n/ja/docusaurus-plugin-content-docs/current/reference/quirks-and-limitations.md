このページは、dappsが考慮すべき動作を文書化します。

## 接続とポップアップ

- 多くのdappホストを接続できます。
- 各ホストが同時に持てるアクティブな接続済みアカウントは1つです。
- ブロッキング承認ポップアップは直列化され、キューに入れられます。
- 重複したアクティブpopup routesは拒否される場合があります。
- ポップアップspamは一時的にブロックされる場合があります。

## UTXOとEVMの分離

- `window.ethereum`はEVM用です。
- `window.pali`はUTXO/Syscoin用です。
- 誤ったチェーンファミリーのメソッドを呼び出すと失敗するか、ネットワーク切り替えが必要になる場合があります。
- UTXO/EVM切り替えにより切断され、再接続が必要になる場合があります。

## EIP-5792ステータス

- `wallet_sendCalls`は実装されています。
- `wallet_getCapabilities`は実装されています。
- `wallet_getCallsStatus`は未サポートのステータスlookupに対して不明なbundle idを返します。
- `wallet_showCallsStatus`はno-op互換メソッドです。

## アトミック性

- Passkeyスマートアカウントは、選択されたバッチ呼び出しを1つのスマートアカウント実行で実行できます。
- 通常のEOAバッチ呼び出しは順次のウォレット送信であり、真のアトミック実行として扱うべきではありません。

## サブスクリプション

`eth_subscribe`と`eth_unsubscribe`は未サポートです。リアルタイムチェーンサブスクリプションには専用のWebSocket RPCプロバイダーを使用してください。

## Passkeys

- Passkeyスマートアカウントサポートは、アクティブチェーンのファクトリー設定に依存します。
- コントラクトデプロイ呼び出しはpasskey `wallet_sendCalls`ではサポートされていません。
- `policyText`はウォレットメタデータおよび表示テキストであり、オンチェーン強制ではありません。
- 必須スポンサーmodeは、スポンサーサービスの可用性とproof検証に依存します。

## Iframes

Paliはトップレベルページにプロバイダーを注入し、iframesには注入しません。
